# Contributing

Thanks for your interest in this project.

## Getting started
1. Fork and clone the repository.
2. Create a virtual environment and install dependencies:
   `pip install -r requirements.txt`
3. Create a feature branch: `git checkout -b feature/your-change`.

## Standards
- Python follows **PEP 8**. Keep functions small, named clearly, and documented.
- Add or update tests under `tests/` for any behavioural change.
- Keep commits focused and write clear commit messages.

## Pull requests
Open a PR against `main` describing what changed and why. Ensure `pytest` passes.
